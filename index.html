Dry Bean Classifier

In this project, I evaluated the effectiveness of various classification models on the UCI Dry Bean Dataset to differentiate between different types of beans, with a specific focus on the 'Sira' bean type, which is hypothetically poisonous. I began by conducting exploratory data analysis (EDA) to gain insights into the features and their distributions. I tested multiple classifiers, including k-Nearest Neighbour, Decision Tree, Logistic Regression, and Random Forest. Performance was evaluated using key metrics such as accuracy, precision, recall, and confusion matrices to visualise the model's effectiveness across all classes. Given the potential risks associated with misclassifying the 'Sira' bean type, I implemented cost-sensitive learning techniques, particularly adjusting the class_weight parameter in the Random Forest classifier and exploring the imbalanced-learn library. The primary goal was to improve the recall for the 'Sira' class to minimise false negatives, ensuring that this bean type was accurately identified without significantly compromising the performance on other classes. The project resulted in a comprehensive analysis comparing the models, discussing the adjustments made to enhance the classification of the 'Sira' bean, and ultimately providing insights into the overall model performance. This project utilised essential libraries such as Scikit-learn and imbalanced-learn to deliver impactful results in the realm of multi-class classification.

Bike Trends London

In this project, I explored patterns in London’s bike ride data to uncover insights related to ride frequency, seasonal variations, and the influence of weather conditions. Using the **london_merged.csv** dataset sourced from Kaggle, I began by loading the data into a Python Jupyter Notebook and performing thorough exploratory data analysis (EDA) to understand the features and their distributions. I then created dynamic visualisations using Tableau to illustrate my findings. Key visualisations included a moving average line chart to depict ride trends over time, heatmaps exploring the relationship between temperature and wind speed, and bar charts showing the distribution of rides across different weather conditions and hourly intervals. The project's main objective was to demonstrate my capabilities as a data analyst through effective data cleaning, visualisation, and storytelling. By analysing ride behaviour in relation to seasonal and weather patterns, I provided actionable insights that could inform decision-making for bike rental services. This project highlighted my proficiency in using data analysis tools and techniques to derive meaningful conclusions from real-world datasets.

Runner Fatigue Analysis

In this project, I classified accelerometer time-series data from recreational runners to detect fatigue in strides, using a dataset with strides labelled as fatigued (F) and non-fatigued (NF) for two subjects, A and B. This analysis involved evaluating baseline and advanced models to detect fatigue accurately. I began with a baseline accuracy assessment by applying a logistic regression model using an SGDClassifier on the raw data for subject A. To enhance classification accuracy, I implemented the Rocket (Random Convolutional Kernel Transform) transformer with an SGDClassifier, using the sktime toolkit for improved performance. Further exploration involved testing alternative classifiers and normalisation methods to improve baseline accuracy. I also tuned the Rocket transformation by experimenting with kernel numbers and classifiers. Lastly, I examined the model’s generalisability by applying the top-performing models from subject A to subject B, comparing the results of personalised versus global models. Requiring libraries such as sktime, scikit-learn, pandas, and matplotlib, this project provided a comprehensive analysis of fatigue detection, offering insights into model adaptability across subjects with time-series classification techniques.
